# auto-custom-rag
**超柔軟なモジュール着せ替え型 RAGエンジン**

---

## 📖 概要

**auto-custom-rag** は、柔軟なモジュール構成を持つRetrieval Augmented Generation（RAG）パイプラインエンジンです。  
設定ファイル（config）を書き換えるだけで、データ取り込みから検索、プロンプト生成、出力整形までのパイプライン構成を自由に変更できます。

- モジュール完全独立設計
- 動的importによる拡張性
- configファイル駆動型パイプライン
- 将来的には生成AIによるパイプライン最適化にも対応予定

---

## 🚀 インストール

必要なパッケージをインストールします。

```bash
pip install -r requirements.txt
```

または

```bash
pipenv install
```

---

## 🛠 使い方

テスト用パイプラインを実行するには：

```bash
python test_run.py
```

設定ファイル (`config_for_test.py`) を編集することで、  
使用するモジュールを自由に組み替えることができます。

---

## 🏗 モジュール構成

以下のモジュールカテゴリに分かれています：

- **loaders**（データ読み込み）
- **parsers**（データ解析）
- **normalizers**（正規化処理）
- **chunkers**（チャンク分割）
- **embedders**（ベクトル変換）
- **retrievers**（検索）
- **filters**（フィルタリング）
- **rerankers**（リランキング）
- **prompts**（プロンプト生成）
- **postprocessors**（出力後処理）
- **models**（モデル呼び出し）

各モジュールは動的に読み込まれ、登録されます。

---

## 🧠 設計思想

- **モジュール完全独立型**  
  各モジュールは単一責任原則に従い、最小限のインターフェースを持ちます。
  
- **動的import設計**  
  コード修正なしで新しいモジュールを追加可能。

- **config駆動型**  
  設定ファイルを編集するだけでパイプラインを着せ替え。

- **将来的な自律化設計**  
  生成AIがタスクやドメインに応じて、最適なパイプラインを提案・構成できる世界を見据えています。
